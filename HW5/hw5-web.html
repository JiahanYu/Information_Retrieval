Homework #5 » Web Search Engine for Computer Science

This homework is an alternative assessment (in place of the exam) for the exchange students.

Commonalities with Homeworks #2 and #3

The indexing and query commands will use an (almost) identical input format to Homeworks #2 and #3, so that you need not modify much of your code to deal with command line processing. To recap:

Indexing: $ python index.py -i directory-of-documents -d dictionary-file -p postings-file

Searching: $ python search.py -d dictionary-file -p postings-file -q query-file -o output-file-of-results

The minor difference from Homeworks #2 and #3 is that query-file specifies a single query instead of a list of queries.

However, significantly different from the previous homework, you will need to collect a set of webpages on Computer Science topics for your system.

Problem Statement: Given 1) a set of webpages on Computer Science topics (to be collected on your own) as the candidate document collection to retrieve from, and 2) a set of queries, return the list of the IDs and the URLs of the relevant documents for each query, in sorted order of relevance. Your search engine should return the entire set of relevant documents (don't threshold to the top K relevant documents).

Your system should return the results for the query query-file on multiple lines. Each line contains the ID and the URL of the document, separated by a single space ' '. Return an empty line if no documents are relevant.

For this assignment, no holds are barred. You may use any type of preprocessing, post-processing, indexing and querying process you wish. You may wish to incorporate or use other python libraries or external resources. However, for python libraries, you'll have to include them with your submission properly -- We will not install new libraries to grade your submissions. Moreover, more credits will be given if you are implementing the features on your own instead of using existing libraries / resources. Please indicate clearly in your documentation which part is implemented by you and which part is done using existing libraries / resources.

Techniques for Retrieval

You are required implement PageRank, as well as one other technique (e.g., Zones and Fields or Query Refinement) in your system. Document the details of your implemenation and evaluation of the techniques in your report.

Nevertheless, do focus on the textual content of the webpage. There is no need to deal with the images, videos and scripts.

More details on the Queries and the Documents

Queries:

The queries are free text queries, such as pagerank examples and "information retrieval" textbook. The keywords enclosed in double quotes are meant to be searched as a phrase. The phrases in the queries are 2 or 3 words long, max; so you if you are able to deal with phrasal queries, you can support them using n-word indices or with positional indices.

Documents:

You should collect 2000 webpages on Computer Science topics, possibly using the crawling algorithm introduced in the lecture.

The files in the collection should be named by their document IDs, which should be an integer between 1-2000.

Since you are required to implement PageRank, you should make sure that the webpages collected are mostly linked (so that the graph is not full of isolated nodes).

What to turn in?

For the system, you are required to submit index.py, search.py, dictionary.txt, postings.txt and a few sample query files (e.g., q1.txt) for running your system.

You do not need to include documentation collection but you should make it available for downloading until the end of May.

For the documentation, you are required to submit README.txt and other supplementary documentations (if any).
